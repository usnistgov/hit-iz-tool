Description

The adverse reaction to the Influenza vaccination of rash within 14 days of dose is reported to the Immunization Registry using a Z22/VXU message.


Comments

No Comments


Pre-condition

An adverse reaction to the Influenza vaccination of rash within 14 days of dose is recorded in the EMR.


Post-Condition

The adverse reaction has been transmitted to the IIS.


Test Objectives

Transmit Standard Patient Immunization History Report: The EHR or other clinical software system directly or indirectly through an intermediary creates and transmits a report of a patient's newly administered or newly identified immunization history to public health immunization registries.

Identify Adverse Event: The EHR or other clinical software system enables capture of structured data regarding adverse events.

Notify Public Health Immunization Registry (IIS) of Update from Adverse Event: The EHR or other clinical software system notifies the public health immunization registry (IIS) of an update due to an adverse event.

Link Standard Codes to Immunization Data: The EHR or other clinical software system links standard codes (i.e., LOINC for lab tests or evaluation tools, SNOMED CT for conditions or observations, NDC codes for current immunizations, CVX for historical immunizations, appropriate codes for administration site, route, method, etc.) to discrete data elements associated with an immunization.

Note: Testing for NDC codes, CVX for immunizations.



Evaluation Criteria

The VXU/Z22 message passes validation using the NIST Immunization VXU Validation Tool (Z22) (context-free). The content of the message correctly reflects the test data (context-based) in accordance with the Test Data Specification and the Message Content.

The test data anticipates that only the updated vaccine is included in the message transmitted to the registry. Due to variation in practice, it is acceptable for the message to include only vaccinations with updated content or a broader vaccination history. The tester should document instances where the updated message includes additional vaccine information.


Notes

The tester verifies that the message includes 'U' in RXA 21 for the Influenza vaccine.

The test data anticipates that the only the updated vaccine is included in the message transmitted to the registry. Due to variation in practice, it is acceptable for the message to include only vaccinations with updated content or a broader vaccination history.The tester should document instances where the updated message includes additional vaccine information.

All NDC coded values are required to use the 11-Character format that includes dashes (‘-‘).
